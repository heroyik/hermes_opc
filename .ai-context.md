# 🧠 AI Development Context: Hermes Book

This file is designed for the Local LLM (Ollama/Hermes) to understand the project's technical stack and design philosophy during flight mode.

## 🛠 Tech Stack
- **Framework**: Next.js 16 (App Router)
- **Styling**: Tailwind CSS 4 + Lucide Icons
- **Motion**: Framer Motion
- **Content**: MDX (`next-mdx-remote/rsc`)
- **Theme**: Deep Space Black (High Contrast, Glassmorphism)

## 📁 Key File Map
- `src/app/book/[slug]/page.tsx`: Main reader logic. Uses `getChapterContent`.
- `src/lib/mdx.ts`: MDX compiler utility.
- `src/content/`: MDX files for each chapter.
- `config.yaml`: Structural metadata (Navigation, Theme).
- `src/app/globals.css`: Design system and custom animations.

## 🎨 Design Rules
1. **Glassmorphism**: Use `bg-white/5 backdrop-blur-xl border-white/10`.
2. **Typography**: Titles are `font-black`, body is `text-secondary/90`.
3. **Colors**: Primary is `#00F0FF` (Cyber Blue), Background is `#050505` (Deep Black).

## 🚀 Development Tips for AI
- When adding new chapters: Create a file in `src/content/[slug].mdx` and add it to `config.yaml`.
- When adding components: Register them in `src/lib/mdx.ts` under the `components` object in `compileMDX`.
- The app uses `output: 'export'` in `next.config.ts` for static GitHub Pages deployment. Do not use dynamic server-side features that break static export.
